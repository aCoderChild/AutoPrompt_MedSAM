# nnU-Net (2D): Automated Medical Image Segmentation Framework

This folder contains inference scripts for nnU-Net, an automated deep learning framework that self-configures based on a given dataset for medical image segmentation tasks.

## Overview

nnU-Net is a self-configuring method that automatically designs and trains a U-Net-based architecture. It requires no manual architecture design or hyperparameter tuning, making it an ideal "out-of-the-box" baseline for 2D medical image segmentation.

## Prerequisites

Install nnU-Net:

```bash
pip install nnunetv2
```

For detailed configuration instructions, see [nnU-Net Documentation](https://github.com/MIC-DKFZ/nnUNet/blob/master/documentation/installation_instructions.md)

## Key Features

- **Automatic Architecture Design**: Configures itself based on data
- **Self-supervised Learning**: Builds from unlabeled data if available
- **Robust Default Settings**: Works well across diverse medical imaging tasks
- **2D Specialization**: This folder focuses on 2D medical images
- **Post-processing**: Automatic test-time augmentation

## Training

Training with nnU-Net requires converting data to the nnU-Net format and running:

```bash
nnUNetv2_train <dataset_id> 2d all
```

For more details on training, see the [nnU-Net GitHub](https://github.com/MIC-DKFZ/nnUNet)

## Inference

### 2D Images

```bash
python infer_nnunet_2D.py \
    -checkpoint nnUNet_results/Dataset001_YourDataset \
    -data_root /path/to/input \
    -pred_save_dir /path/to/output \
    --save_overlay \
    -png_save_dir /path/to/overlay \
    -num_workers 2
```

**Options:**
- `--grey`: Use this flag for grayscale images
- `--save_overlay`: Save overlay visualizations
- `-num_workers`: Number of parallel workers

### For Grayscale Images

```bash
python infer_nnunet_2D.py \
    -checkpoint nnUNet_results/Dataset002_X-Ray \
    -data_root /path/to/input \
    -pred_save_dir /path/to/output \
    --grey \
    --save_overlay \
    -png_save_dir /path/to/overlay
```

## Data Format

Input data should be in the `npz` format generated by MedSAM's preprocessing pipeline.
