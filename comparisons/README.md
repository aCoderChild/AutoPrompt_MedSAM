# Comparison Models for Medical Image Segmentation

This folder contains implementations of various medical image segmentation models organized by category. Each model includes training and inference scripts for both 2D and 3D medical images, with documentation and usage examples.

## Overview

The models are organized into the following categories:

### 1. **Standard CNN Models**

#### [U-Net](./U-Net/)
- Classic encoder-decoder architecture with skip connections
- Simple yet powerful baseline for medical image segmentation
- **Files**: `train_unet.py`, `infer_unet_2D.py`, `infer_unet_3D.py`

#### [DeepLabV3+](./DeepLabV3+/)
- Atrous spatial pyramid pooling for multi-scale context
- Effective semantic segmentation with ResNet backbone
- **Files**: `train_deeplabv3_res50.py`, `infer_deeplabv3_res50_2D.py`, `infer_deeplabv3_res50_3D.py`

---

### 2. **Transformer/Hybrid Models**

#### [Swin-Unet](./Swin-Unet/)
- Vision transformer-based U-shaped architecture
- Shifted window attention for efficient processing
- **Files**: `train_swin_unet.py`, `infer_swin_unet_2D.py`, `infer_swin_unet_3D.py`

#### [Mask2Former](./Mask2Former/)
- Unified instance, semantic, and panoptic segmentation
- Transformer-based with masked attention
- **Files**: `train_mask2former.py`, `infer_mask2former_2D.py`, `infer_mask2former_3D.py`

---

### 3. **Medical Auto-Framework**

#### [nnU-Net](./nnU-Net/)
- Self-configuring automated deep learning framework
- Requires no manual architecture design or hyperparameter tuning
- **Files**: `infer_nnunet_2D.py`, `infer_nnunet_3D.py`
- *Note: Training uses nnUNetv2 CLI; see README for details*

---

### 4. **Foundation Model**

#### [MedSAM](./MedSAM/)
- Large-scale pre-trained foundation model
- Zero-shot or fine-tuned capability
- Supports various prompt types (bounding boxes, points, masks)
- **Files**: `infer_medsam_2D.py`, `infer_medsam_3D.py`

---

### 5. **Task-Specific Model**

#### [SUMNet](./SUMNet/)
- Supervised Uncertainty-aware Medical Segmentation Network
- Outputs both predictions and uncertainty maps
- Clinically relevant confidence estimation
- **Files**: `train_sumnet.py`, `infer_sumnet_2D.py`, `infer_sumnet_3D.py`

---

## Data Format

All models expect input data in the **npz format** generated by MedSAM's preprocessing pipeline. The data structure should contain:
- `image`: The medical image (2D or 3D array)
- `label`: Ground truth segmentation mask (for training)

### Converting Your Data

See [MedSAM Data Preprocessing](https://github.com/bowang-lab/MedSAM#data-preprocessing) for details on converting medical images to npz format.

## Quick Start

### For 2D Images

1. **U-Net Example:**
```bash
# Training
python U-Net/train_unet.py -i /path/to/data -o ./models/unet

# Inference
python U-Net/infer_unet_2D.py \
    -checkpoint ./models/unet/unet_best.pt \
    -data_root /path/to/test_images \
    -pred_save_dir ./results/unet
```

2. **DeepLabV3+ Example:**
```bash
# Training
python DeepLabV3+/train_deeplabv3_res50.py -i /path/to/data -o ./models/deeplabv3

# Inference
python DeepLabV3+/infer_deeplabv3_res50_2D.py \
    -checkpoint ./models/deeplabv3/deeplabv3plus_best.pt \
    -data_root /path/to/test_images \
    -pred_save_dir ./results/deeplabv3
```

3. **MedSAM Zero-shot:**
```bash
# No training needed! Use pretrained model
python MedSAM/infer_medsam_2D.py \
    -checkpoint /path/to/medsam_vit_b.pt \
    -data_root /path/to/test_images \
    -pred_save_dir ./results/medsam
```

### For 3D Images

Replace the 2D inference scripts with 3D variants:
```bash
python U-Net/infer_unet_3D.py -checkpoint ... -data_root ... -pred_save_dir ...
```

## Common Hyperparameters

Most training scripts support these parameters:

- `-i, --input`: Path to training dataset (npz files)
- `-o, --output`: Output directory for trained model
- `-b, --batch_size`: Batch size (default varies by model)
- `--max_epochs`: Number of training epochs (default: 500)
- `--num_workers`: Data loading workers (default: 4)

Inference scripts support:
- `-checkpoint`: Path to trained model checkpoint
- `-data_root`: Path to test images
- `-pred_save_dir`: Output directory for predictions
- `--save_overlay`: Save visualization overlays
- `-png_save_dir`: Directory for overlay images

## Output Format

- **Predictions**: Saved as npz files with segmentation masks
- **Overlays** (optional): PNG images showing prediction on original
- **Uncertainty Maps** (SUMNet only): Confidence scores for predictions

## Environment Setup

### Install Dependencies

```bash
# PyTorch (adjust CUDA version as needed)
pip install torch torchvision

# Framework-specific packages
pip install monai                              # For U-Net, Swin-Unet
pip install segment-anything                  # For MedSAM
pip install timm                              # For Transformer models
pip install nnunetv2                          # For nnU-Net
pip install segmentation-models-pytorch       # For DeepLabV3+
```

### GPU Requirements

- **Minimum**: 4GB VRAM (U-Net, DeepLabV3+)
- **Recommended**: 8GB+ VRAM (Swin-Unet, Mask2Former, fine-tuned MedSAM)
- **Large models**: 12GB+ VRAM (nnU-Net with large datasets)
