# SUMNet: Ensemble-based Medical Segmentation Network

This folder contains inference scripts for SUMNet, an ensemble learning approach that combines 8 state-of-the-art semantic segmentation architectures for robust medical image analysis.

## Overview

SUMNet is an ensemble learning model that combines predictions from 8 different semantic segmentation architectures:
- **FCN** (Fully Convolutional Networks)
- **UPerNet** (Unified Perceptual Parsing)
- **DNLNet** (Disentangled Non-Local Neural Networks)
- **DeepLabV3** (Atrous Spatial Pyramid Pooling)
- **OCRNet** (Object-Contextual Representations)
- **PSANet** (Point-wise Spatial Attention Network)
- **DANet** (Dual Attention Network)
- **CCNet** (Criss-Cross Attention Network)

This ensemble approach improves accuracy and robustness compared to individual models, particularly effective for adenoid hypertrophy segmentation in nasopharyngoscopy images.

## Architecture Features

- **Ensemble Learning**: Combines 8 diverse segmentation architectures
- **Multi-head Decoder**: Each base model contributes independent predictions
- **Voting-based Consensus**: Predictions aggregated through majority voting or weighted averaging
- **Transfer Learning**: Pre-trained models on large-scale datasets (Cityscapes, ADE20K)
- **Robustness**: Reduced overfitting and improved generalization
- **Stability**: Superior consistency across batch variations

Training is performed using MMSegmentation's CLI tools for each base model.
For detailed training instructions, see [MMSegmentation Documentation](https://github.com/open-mmlab/mmsegmentation)

Example for training individual models:
```bash
python -m mmseg.train <config_file>
```

## Inference

### 2D Images

```bash
python infer_sumnet_2D.py \
    -model_configs /path/to/mmsegmentation/configs \
    -checkpoints /path/to/model/checkpoints \
    -data_root /path/to/input \
    -pred_save_dir /path/to/output \
    --save_overlay \
    -png_save_dir /path/to/overlay
```

### 3D Images (Slice-by-slice)

```bash
python infer_sumnet_3D.py \
    -model_configs /path/to/mmsegmentation/configs \
    -checkpoints /path/to/model/checkpoints \
    -data_root /path/to/input \
    -pred_save_dir /path/to/output \
    --save_overlay \
    -png_save_dir /path/to/overlay
```

## Ensemble Models

SUMNet uses the following 8 base models (in order of performance):

| Rank | Model | Type | Key Feature |
|------|-------|------|-------------|
| 1 | FCN | Fully Convolutional | Dense prediction |
| 2 | UPerNet | Pyramid Parsing | Multi-scale context |
| 3 | DNLNet | Non-local | Long-range dependencies |
| 4 | DeepLabV3 | Atrous Conv | Multi-scale dilation |
| 5 | OCRNet | Contextual | Object context |
| 6 | PSANet | Spatial Attention | Point-wise attention |
| 7 | DANet | Dual Attention | Channel + spatial |
| 8 | CCNet | Criss-cross | Efficient attention |

## Prediction Aggregation

SUMNet combines predictions through:
- **Majority Voting**: For binary segmentation
- **Weighted Averaging**: Weights based on individual model confidence
- **Consensus Filtering**: Soft voting with confidence thresholds

## Output Format

- **Segmentation Mask**: Ensemble consensus predictions
- **Confidence Map**: Average confidence across ensemble
- **Overlay**: Visual overlay of segmentation on original image

## Data Format

Expects input data in `npz` format generated by MedSAM's preprocessing pipeline.

## Advantages

- **Higher Accuracy**: Ensemble outperforms individual models
- **Better Robustness**: Reduced sensitivity to training variations
- **Improved Consistency**: Lower inter-model variance
- **Enhanced Stability**: More reliable for clinical applications
- **Reduced Overfitting**: Generalization benefits from diverse architectures

## Performance Metrics (Adenoid Hypertrophy)

Based on the paper's results on 1500 nasopharyngoscopy images:
- **Precision**: 0.9616 (highest among all models)
- **Recall**: 0.7538
- **MIoU**: 0.8519 (highest among all models)
- **Overall Accuracy**: 0.9243 (highest among all models)
- **Cohen's Kappa**: 0.87 (excellent agreement)
- **AUC**: 0.85 (superior to expert evaluation: 0.74)

## Notes

- Ensemble approach requires all 8 models to be trained/loaded
- Memory intensive: ensure sufficient GPU memory (recommended: 12GB+)
- Inference time: slower than individual models (~8x slower)
- Trade-off: accuracy gain justifies computational cost
