# DeepLabV3+: Atrous Spatial Pyramid Pooling for Semantic Segmentation

This folder contains training and inference scripts for DeepLabV3+, a state-of-the-art semantic segmentation architecture using atrous convolution and spatial pyramid pooling.

## Overview

DeepLabV3+ is a semantic segmentation model that uses atrous convolutions (dilated convolutions) with spatial pyramid pooling to capture multi-scale contextual information. It's highly effective for medical image segmentation with its encoder-decoder structure.

## Prerequisites

Install required dependencies:

```bash
pip install torch torchvision
pip install segmentation-models-pytorch
```

## Architecture Features

- **Atrous Convolution**: Multi-scale receptive fields without losing resolution
- **Spatial Pyramid Pooling**: ASPP captures context at multiple scales
- **Encoder-Decoder**: Feature recovery through decoder module
- **ResNet Backbone**: Powerful feature extraction backbone
- **Efficient Design**: Good balance between accuracy and computation

## Training

To train DeepLabV3+ for medical image segmentation:

```bash
python train_deeplabv3_res50.py \
    -i /path/to/input \
    -o /path/to/output \
    -b 32 \
    --max_epochs 500 \
    --num_workers 4
```

**Parameters:**
- `-i, --input`: Path to input dataset (npz format)
- `-o, --output`: Path to save trained model
- `-b, --batch_size`: Batch size (default: 32)
- `--max_epochs`: Maximum number of epochs
- `--num_workers`: Number of workers for data loading
- `--compile`: Compile model for acceleration (optional)

## Inference

### 2D Images

```bash
python infer_deeplabv3_res50_2D.py \
    -checkpoint /path/to/deeplabv3plus_best.pt \
    -data_root /path/to/input \
    -pred_save_dir /path/to/output \
    --save_overlay \
    -png_save_dir /path/to/overlay \
    -num_workers 2
```

### 3D Images

```bash
python infer_deeplabv3_res50_3D.py \
    -checkpoint /path/to/deeplabv3plus_best.pt \
    -data_root /path/to/input \
    -pred_save_dir /path/to/output \
    --save_overlay \
    -png_save_dir /path/to/overlay \
    -num_workers 2
```

## Data Format

Input data should be in the `npz` format generated by MedSAM's preprocessing pipeline. For details, refer to [MedSAM preprocessing](https://github.com/bowang-lab/MedSAM#data-preprocessing).

## Model Variants

- **ResNet-50**: Good balance between accuracy and speed
- **ResNet-101**: Higher accuracy at cost of computational resources
